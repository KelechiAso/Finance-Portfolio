My Key Functions & What They're For

The tutorial really breaks down how to use a few core functions to build some heavy-duty models. It's not just about knowing them, but knowing when to use them.

Data & Reference Functions

These functions are great for pulling data from one place to another efficiently.

  • VLOOKUP and HLOOKUP: These are like a quick search function. You use them when you need to find something in a big table and grab a related piece of info. The tutorial showed me they're great for pulling data like prices or dates from a separate sheet of a financial model.
  • INDEX and MATCH: This is the upgrade to VLOOKUP. It's more flexible because you can search for data vertically or horizontally. You use `MATCH` to find the location of an item, and then `INDEX` to pull the actual data from that spot. It's way more powerful.
  • OFFSET: This returns a reference to a range that's a specified number of rows and columns away from a starting point. It's super powerful for creating dynamic charts and tables that automatically update as your data grows.

Financial Functions

These are the real finance tools that do the heavy lifting in valuation and modeling.

  • NPV (Net Present Value): Calculates the value of a series of future cash flows in today's dollars. It's one of the most fundamental tools for deciding whether to invest in a project.
  • IRR (Internal Rate of Return): This tells you the return you're getting on an investment. If the `IRR` is higher than your cost of capital, the investment is usually a good idea. It's a key function for capital budgeting.
  • PMT (Payment): This calculates the payment for a loan based on constant payments and a constant interest rate. Essential for building debt schedules in a financial model.
  • RATE: This function returns the interest rate for an annuity. It's used to solve for an unknown interest rate when you have all the other variables.

Statistical & Logical Functions

These functions help build smart, dynamic, and error-proof models.

  • IF, AND, OR: Used to build smart models that can make decisions. For example, you can use an `IF` statement to calculate a tax rate only if a company is profitable, making the model dynamic.
  • SUMIFS and COUNTIFS: These functions let you calculate sums or counts based on multiple criteria. This is a must-have for detailed financial reporting and analysis.
  • Geometric Mean (GEOMEAN): This is a special type of average that's more accurate for calculating investment returns because it accounts for compounding. The tutorial explained that it gives a better picture of performance over time than a simple average.
  • Extrapolation: This is a key financial modeling skill. Using functions like `FORECAST.LINEAR` and other logic, you can predict future values based on past trends, which is a core part of forecasting revenue and expenses.

Steps for Building a Financial Model in Excel

The video laid out a clear process for building a financial model from scratch. It's not just about the formulas; it's about the workflow.

  1. Get the Data: You start by grabbing all the historical financial data you can find. This includes the Income Statement, Balance Sheet, and Cash Flow Statement.
  2. Organize Your Sheets: Don't just throw everything in one place. The tutorial teaches you to use separate tabs for inputs, assumptions, calculations, and the final output. This keeps everything clean and easy to follow.
  3. Input & Forecast: You use the historical data to set up your assumptions. Then, you use those assumptions to project the three financial statements into the future. This is where most of the work happens.
  4. Connect Everything: This is the most important step. You have to link all three statements together. For example, Net Income from the Income Statement has to be linked to Retained Earnings on the Balance Sheet. If you mess this up, the whole thing breaks.
  5. Check for Errors: After everything is linked, you audit the model to make sure the Balance Sheet actually balances. If assets don't equal liabilities plus equity, something is wrong.
